Quick Start

One line downloads, sets up, starts, and opens Haven Desk.

This is the low-friction pilot path. It downloads the public repo, runs the existing setup, starts the local stack, and opens localhost:4141.

pilot

# Installs Homebrew/Git if needed, downloads Haven Desk, starts the local app.

curl -fsSL https://omerakben.github.io/haven-desk/install.sh | bash

# Run in PowerShell. Installs Git if needed, downloads Haven Desk, starts the local app.

irm https://omerakben.github.io/haven-desk/install.ps1 | iex
Downloads to ~/HavenDesk by default. Requires Docker Desktop and Ollama because the app runs local-first.

Honest positioning

This is not trying to beat Open WebUI at being an AI platform.

Open WebUI is broader and more powerful for technical users. Haven Desk should win a narrower job: small-business operators who want a local desk that starts from daily admin work, gives them reviewed next steps, and keeps the workflow simple.

Haven Desk Workflow-first local desk

Quick actions, packs, tasks, drafts, and review-before-use patterns for repeat admin jobs.

Open WebUI General AI interface

Excellent for people who already know how to configure models, prompts, tools, and workflows.

Why buy? Less setup, less thinking, more follow-through

The pitch is not more AI. It is a calmer way to finish the admin work small businesses keep dropping.

What the installer does

The one-liner is thin on purpose.

It avoids inventing a second install system. The hosted script downloads the repo, then runs the same setup and launch commands we already validate locally.

01

Clone or update https://github.com/omerakben/haven-desk.git into ~/HavenDesk.

02

Run ./swiss setup on macOS or .\swiss.ps1 setup on Windows.

03

Start Docker Desktop when needed, then run the existing up command.

04

Open Haven Desk at http://localhost:4141; Open WebUI remains available on 4142.
